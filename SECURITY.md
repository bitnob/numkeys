# Security Policy

## Supported Versions

NumKeys is currently pre-release software. Security fixes are applied on the default branch.

## Reporting a Vulnerability

Do not open public issues for potential vulnerabilities.

Please report security issues privately with:

- a clear description of the issue
- impact assessment
- reproduction steps or proof of concept
- affected components and versions/commit hashes

Use the repository security reporting channel on GitHub:

- GitHub Security Advisories (preferred)

If advisories are unavailable, open a private maintainer contact through the organization profile.

## Disclosure Process

1. We acknowledge receipt as quickly as possible.
2. We validate and triage severity.
3. We prepare and test a fix.
4. We coordinate disclosure timing.
5. We publish remediation guidance and affected version details.

## Scope Notes

- `numkeys-core` protocol correctness and signature verification are high-priority surfaces.
- Reference services (for example `issuer-node`) are in scope for implementation-level vulnerabilities.
